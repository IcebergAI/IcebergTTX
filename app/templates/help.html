How to run tabletop exercises with IcebergTTX.

Roles
Facilitator · Participant · Observer
Workflow
Create → Run → Review
Scenarios
JSON format & branching

Roles

Facilitator

Runs the exercise. Creates and imports scenarios, starts exercises, adds participants, and controls the inject feed — deciding when each inject is released and which branch to follow. Can inject simulated inbound communications, view all participant responses and team comments, and export transcripts.

Participant

Receives injects as they are released, adds team comments, and submits responses — choosing a stance (if options are provided) and writing free-text reasoning. Sees only the injects and comments assigned to their team. Cannot see other participants' formal responses.

Observer

Watches the exercise in read-only mode. Sees all injects but cannot submit responses. Useful for senior stakeholders, auditors, or training evaluators.

Workflow

1
Create or import a scenario

Go to Scenarios → New scenario to build an inject tree in the editor, or use Import JSON to load a pre-built scenario file. Each inject has a title, content, optional target teams, and branch options that point to the next inject. See Scenario format for the JSON schema.

2
Create an exercise

Go to Exercises → New exercise, give it a title, and select a scenario. Optionally enable LLM assessment to have Claude evaluate each response and suggest follow-up injects (requires ANTHROPIC_API_KEY in your .env).

3
Add participants

Open the facilitator console and use the Participants panel (right pane) to search for registered users and enrol them. Each participant is assigned a team. Share the exercise link /exercises/{id}/participate with participants, or invite them to the /register page first.

4
Start the exercise and release injects

Hit Start from the exercises list or the facilitator console. Injects appear in the left pane as pending nodes. Press Release to push an inject to participants — they'll receive it instantly via WebSocket. Once they respond, review the responses and team comments in the middle pane and choose which branch to release next. You can Pause the exercise at any point to halt new submissions.

5
Inject communications

Open Communications from the sidebar. As facilitator, click Inject inbound to simulate an incoming message from an external entity (ICO, NCSC, CEO, etc.) targeted at specific teams. Participants see these in their own Communications inbox. Outbound messages generated automatically by triggered communications in the scenario definition also appear here.

6
Complete and export

When all injects are resolved, click Complete to close the exercise. From the facilitator console's right pane, export the full transcript (JSON), responses (CSV), or AI assessments (JSON) for post-exercise review and reporting.

Scenario format

Scenarios are imported and exported as JSON. The top-level structure:

scenario.json
{
  "schema_version": "1.0",
  "title": "Ransomware incident",
  "description": "A simulated ransomware attack affecting core infrastructure.",
  "tags": ["cyber", "ransomware"],
  "participant_teams": [
    { "id": "it_ops", "label": "IT Operations" },
    { "id": "legal",  "label": "Legal & Compliance" }
  ],
  "start_inject_id": "inject_01",
  "injects": [ ... ],
  "debrief_notes": "Key learning: notify ICO within 72 hours."
}

Each inject in the injects array:

{
  "id": "inject_01",
  "title": "Ransomware detected",
  "content": "SOC has detected encrypted files on 3 servers...",
  "target_teams": ["it_ops"],
  "free_text_response": true,
  "sequence_order": 1,
  "next_inject_id": null,
  "options": [
    { "id": "opt_isolate", "label": "Isolate affected systems immediately", "next_inject_id": "inject_02a" },
    { "id": "opt_monitor", "label": "Monitor and gather more information",  "next_inject_id": "inject_02b" }
  ],
  "triggers_communications": [
    {
      "direction": "inbound",
      "external_entity": "NCSC",
      "subject": "Ransomware advisory",
      "body": "We have been made aware of a campaign targeting...",
      "delay_seconds": 120,
      "visible_to_teams": ["it_ops"]
    }
  ]
}
target_teams
IDs from participant_teams. Blank array means all teams receive the inject.
options
Branch choices shown to participants. next_inject_id: null ends the branch.
next_inject_id
Linear continuation for injects without branch choices. Participants submit a free-text response, then facilitators can release the next inject.
triggers_communications
Messages automatically injected into the comms inbox when this inject is released. delay_seconds staggers delivery.

LLM assessment

When enabled on an exercise, Claude evaluates each participant response and produces:

  • A decision quality rating — strong, acceptable, or poor
  • A brief assessment of the reasoning
  • A suggested follow-up inject the facilitator can approve and queue

Set ANTHROPIC_API_KEY in your .env file to enable. Assessments appear in the AI assessment column of each response card in the facilitator console, and in the right pane's AI suggestions widget.

Tips

The sidebar Communications link opens the inbox for the currently active exercise. If no exercise is live, it opens a short empty state with a path back to Exercises.
A paused exercise disables participant response and comment submission but keeps the WebSocket connection alive — participants see the yellow "Paused" banner and forms become read-only.
Scenarios can be exported from the scenario detail page and re-imported into a different IcebergTTX instance — useful for sharing scenario packs between teams.
The inject tree editor validates that all next_inject_id references exist. The scenario detail page also shows a live validation sidebar — fix any issues before running an exercise.
Use the Filter tabs in the facilitator response feed (All / Current node / Flagged) to focus on responses that need attention. "Flagged" shows responses the AI rated as poor.
